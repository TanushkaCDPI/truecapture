TrueCapture implements the C2PA open standard — the same cryptographic provenance protocol used by Adobe, Google, Sony, the BBC and Google Pixel. A SHA-256 hash of your media is signed with an ECDSA key at capture time, embedded as a C2PA manifest in the file itself, and the public key is registered on DeDi.global — a decentralised public key directory. Verification is stateless and fully client-side. If a single pixel changes, the signature breaks.

How it works

1

Capture

Take a photo, record a video, or go live on screen or camera using the TrueCapture Chrome extension or mobile web app.

2

Sign

Your media is instantly signed with a cryptographic key using the C2PA standard. The signature is embedded in the file itself. Public key registered on DeDi.global.

3

Share

Paste the verify link in your caption or message. Anyone who sees it taps the link for an instant verdict — no app needed.
